[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Here's how it works under the hood using ADK 2.0 Workflows. When a user first connects, our intake node uses a Human-in-the-Loop pause to explicitly gather their career context. That context then flows through our security checkpoint, which scrubs sensitive PII like phone numbers and blocks any attempts to prompt-inject or ask for help falsifying credentials.

Once cleared, the request hits our Orchestrator agent. Based on the user's need, the Orchestrator delegates to three specialized sub-agents: the resume_builder, the interview_coach, and the job_matcher. These sub-agents use AgentTool to communicate and pull live data from our JobReady MCP Server.

[3:00 — SECURITY & MCP]
Notice how the agent is entirely grounded by the MCP Server. We aren't relying on the LLM to hallucinate job salaries or interview questions; those come from our domain tools. And earlier, if I had pasted my real phone number or SSN into that intake form, the security checkpoint would have automatically redacted it to protect my privacy.
